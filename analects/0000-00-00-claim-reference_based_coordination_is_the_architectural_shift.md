# Reference-based coordination is the architectural shift

## Retrieval Gist

What practical interoperability change is Mycite aiming for?

## Claim

File exports, spreadsheets, JSON payloads, and other shipped artifacts solve transmission, not interoperability. Each receiver still has to infer meaning, map fields, and reconcile updates against local state. A more durable model coordinates through references, declared semantics, provenance, and retrievable context so that systems can resolve shared objects and incremental changes without repeated file shipping. That is the architectural shift: from opaque artifact exchange to interpretable reference-based coordination.

## Boundaries

- This does not claim files disappear from the workflow.
- It does not eliminate mapping work; it makes that work explicit and reusable.
